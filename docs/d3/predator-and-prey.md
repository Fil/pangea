```js
md`# Predator and Prey

Imagine an island of cats üêà and mice üêÅ. If mice are plentiful, the cats eat well and reproduce rapidly. But as the cats multiply, they eat more mice. Soon the mouse population is decimated and the cats begin to starve. With fewer cats, the mice recover, and the cycle repeats.

The [Lotka‚ÄìVolterra equations](https://en.wikipedia.org/wiki/Lotka‚ÄìVolterra_equations) model such a dynamic predator‚Äìprey interaction. The model is a pair of (ordinary differential) equations where üêÅ is the number of prey and üêà is the number of predators:

${tex.block`\begin{aligned}
\frac{\mathrm{d}üêÅ}{\mathrm{d}t} &= \alpha üêÅ-\beta üêÅüêà \\[2ex]
\frac{\mathrm{d}üêà}{\mathrm{d}t} &= \delta üêÅüêà-\gamma üêà
\end{aligned}`}

In this model, the prey have unlimited food and grow exponentially (${tex`\alpha`}); however, they are sometimes eaten (${tex`\beta`}) by predators. The predators have unlimited appetite, but eat only prey, limiting their growth (${tex`\delta`}); their decay is exponential (${tex`\gamma`}). The parameters ${tex`\alpha`}, ${tex`\beta`}, ${tex`\gamma`}, and ${tex`\delta`} thus describe how the two populations behave, while ${tex`\tfrac{\mathrm{d}üêÅ}{\mathrm{d}t}`} and ${tex`\tfrac{\mathrm{d}üêà}{\mathrm{d}t}`} are the rates at which each population is increasing or decreasing.

By iteratively evaluating these equations‚Äîby starting with some initial populations, calculating the rates, adjusting the populations accordingly, then repeating this process many times‚Äîwe can compute the populations over time as shown in the plot below. Drag the cat üêà or the mouse üêÅ to change the initial populations. (You can‚Äôt have fractional mice, so interpret the numbers as thousands or similar.)`;
```

```js
md`
This cycle suggests a counterintuitive but plausible result: if you instantaneously reduce the number of prey, predators will decline, allowing prey to recover, and a (temporarily) _larger_ prey population than you started with. More generally, feedback loops in dynamical systems can make it hard to predict what will happen when you artificially introduce change!

Interacting with the phase plot reveals something else: see if you can adjust the initial populations <svg width=8 height=16><circle r=3.5 cx=4 cy=10></circle></svg> to contract the cycle <svg width=16 height=16><circle r=7 cx=8 cy=8 fill=none stroke=#000 stroke-opacity=0.25 stroke-width=1.5></circle></svg> down to a single point. What does this singularity mean in terms of predator‚Äìprey behavior? It is _equilibrium_: both populations are unchanging; both predator and prey are dying at the same rate they are reproducing.

We can see the stationary point directly‚Äîno interaction required‚Äîby plotting the entire vector field (the rates of change for _all possible_ conditions). It‚Äôs like a wind map: if you set the initial populations to a given point, the populations will follow the lines of the field and eventually loop back to the same point. The stationary point is the eye of the storm.
`;
```

```js
md`This plot also represents speed with color‚Äîthe angry red at the top indicates conditions where the populations are changing rapidly. The farther you move from the stationary point, the wilder (and less realistic) the oscillations. (We should be skeptical drawing conclusions from extreme scenarios in simple models.)

We can also derive the stable point algebraically by noting that the rates of change ${tex`\tfrac{\mathrm{d}üêÅ}{\mathrm{d}t}`} and ${tex`\tfrac{\mathrm{d}üêà}{\mathrm{d}t}`} are zero at equilibrium:

${tex.block`\begin{aligned}
0 &= \alpha üêÅ - \beta üêÅüêà \\
0 &= \delta üêÅüêà - \gamma üêà
\end{aligned}`}

Equivalently,

${tex.block`\begin{aligned}
\alpha üêÅ &= \beta üêÅüêà \\
\gamma üêà &= \delta üêÅüêà
\end{aligned}`}

And thus,

${tex.block`\begin{aligned}
üêà &= \frac{\alpha}{\beta} \\[2ex]
üêÅ &= \frac{\gamma}{\delta}
\end{aligned}`}
`;
```

```js echo
const simulate = {
  const l = 24, k = 20, n = l * k;
  const solver = new odex.Solver(2);
  solver.denseOutput = true;
  return ({alpha, beta, gamma, delta, x0, y0}) => {
    let i = 0;
    const T = new Float64Array(n);
    const X = new Float64Array(n);
    const Y = new Float64Array(n);
    const input = (_, [x, y]) => [alpha * x - beta * x * y, delta * x * y - gamma * y];
    const output = solver.grid(1 / k, (t, [x, y]) => (T[i] = t, X[i] = x, Y[i] = y, ++i));
    solver.solve(input, 0, [x0, y0], l, output);
    return [T, X, Y];
  };
}
```

```js echo
function cycle(T, X, Y) {
  let j,
    x0 = X[0],
    x1 = X[1],
    x2;
  for (let i = 2, n = T.length; i < n; ++i) {
    (x0 = x1), (x1 = x2), (x2 = X[i]);
    if (x0 > x1 && x1 < x2) {
      if (j > 0) return [T.slice(j, i), X.slice(j, i), Y.slice(j, i)];
```

```js echo
class View {
  constructor(value) {
    this._list = [];
    this._value = value;
  }
  get value() {
    return this._value;
  }
  set value(value) {
    this._value = value;
    this.dispatchEvent({type: "input", value});
  }
  addEventListener(type, listener) {
    if (type != "input" || this._list.includes(listener)) return;
    this._list = [listener].concat(this._list);
  }
  removeEventListener(type, listener) {
    if (type != "input") return;
    this._list = this._list.filter((l) => l !== listener);
  }
  dispatchEvent(event) {
    const p = Promise.resolve(event);
    this._list.forEach((l) => p.then(l));
  }
}
```
